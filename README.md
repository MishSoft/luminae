# Luminae Project Review & Action Plan

ეს დოკუმენტი წარმოადგენს პროექტის არსებული მდგომარეობის შეფასებას და მოიცავს რეკომენდაციებს იმის შესახებ, თუ რა უნდა გასწორდეს და რა მიდგომები უნდა გამოიყენოთ კოდის გასაუმჯობესებლად.

## 1. ფაილების სტრუქტურა და SEO მეტა თეგები
- **SEO-ს პრობლემა**: `app/layout.tsx`-ში ამჟამად დარჩენილია Next.js-ის სტანდარტული მეტა მონაცემები (title: "Create Next App", description: "Generated by create next app"). 
  - **როგორ გავასწოროთ**: უნდა შეიცვალოს ბრენდის შესაბამისი title და description-ით. სასურველია დაემატოს `OpenGraph (og:)` და `Twitter` თეგები სოციალურ ქსელებში სწორად გაზიარებისთვის.
- **ფაილების სტრუქტურა**: პროექტში გვაქვს როგორც `app/page.tsx`, ასევე ცარიელი და აზრს მოკლებული Route ჯგუფი: `app/(main)/page.tsx` და `app/(main)/layout.tsx`. ეს იწვევს სტრუქტურულ გაურკვევლობას. 
  - **როგორ გავასწოროთ**: უნდა ჩამოყალიბდეთ ერთიან სტრუქტურაზე: ან წაიშალოს `app/page.tsx` და მთავარი გვერდი გადავიდეს `(main)` ჯგუფში, ან პირიქით, წაიშალოს `(main)` ფოლდერი.

## 2. ვიზუალური მსგავსება დიზაინთან და სტილები
- **Tailwind კლასები**: Header-ის კომპონენტებში (`MainHeader.tsx`, `SubHeader.tsx`) Tailwind კლასები საკმაოდ გადატვირთულია.
  - **როგორ გავასწოროთ**: კოდის წაკითხვადობის გასაზრდელად შეგიძლიათ გამოიყენოთ `tailwind-merge` და `clsx` ბიბლიოთეკები.
- **სურათების გზები (Paths)**: `HeaderPromo.tsx`-ში Image კომპონენტის `src` არის `'assets/images/...'`. 
  - **როგორ გავასწოროთ**: სასურველია გზა იწყებოდეს `/`-ით (`'/assets/images/...'`), რათა ნებისმიერ გვერდზე სურათის ჩატვირთვა იყოს გარანტირებული მარშრუტის სიღრმის მიუხედავად.

## 3. ფუნქციონალის მსგავსება მოცემულ დავალებასთან
- **მართლწერა**: `MainHeader.tsx`-ში Search Input-ის placeholder-ში არის მექანიკური შეცდომა (`"Search Producs"`), უნდა იყოს `"Search Products"`.
- **Dropdown კომპონენტი**: `DropDown.tsx` ვიზუალურად მუშაობს, მაგრამ აკლია Accessibility (a11y). 
  - **როგორ გავასწოროთ**: უნდა დაემატოს `ARIA` ატრიბუტები და კლავიატურით ნავიგაციის მხარდაჭერა (მაგალითად: `Enter/Space` გაღებისთვის, `Esc` დახურვისთვის, `Arrow` კლავიშები მენიუში ნავიგაციისთვის).
- **ჰარდკოდირებული მნიშვნელობები**: Currency და Language (`SubHeader.tsx`), ასევე Cart/Favorites მრიცხველები (`HeaderUserSide.tsx`, `<span ...>3</span>`) გაწერილია პირდაპირ (hardcoded).
  - **როგორ გავასწოროთ**: ეს მონაცემები უნდა მოდიოდეს გლობალური state-დან ან პროპსებიდან (მაგ. Zustand, Redux ან Context API გამოყენებით).

## 4. არსებული ფუნქციონალის პერფომანსი და სუფთა კოდი
- **Event Listener-ები**: `DropDown.tsx`-ში `mousedown` ლისენერი ემატება კომპონენტის ჩატვირთვისთანავე (`useEffect`-ში ცარიელი დეფენდენსით). 
  - **როგორ გავასწოროთ**: უკეთესი პერფომანსისთვის ლისენერი უნდა ემატებოდეს მხოლოდ მაშინ, როცა დროფდაუნი ღიაა (`isOpen === true`).
- **კოდის გამეორება (DRY პრინციპი)**: Icon კომპონენტების (`CardIcon.tsx`, `HeartIcon.tsx`, ა.შ.) უმეტესობა მსგავსია. 
  - **როგორ გავასწოროთ**: შესაძლებელია ერთიანი უნივერსალური `Icon` კომპონენტის შექმნა, რომელიც მიიღებს შესაბამის SVG path-ს პროპსად.

## 5. დამატებითი Feature-ი (სურვილისამებრ)
- **i18n (მრავალენოვნება)**: რადგან Header-ში გაქვთ ენის გადამრთველი (English/Georgian), სასურველია დაემატოს რეალური ლოკალიზაცია (`next-intl`-ის გამოყენებით).
- **Dark Mode / თემები**: Next.js-სა და Tailwind-ში მარტივად ინტეგრირდება `next-themes`, რაც საშუალებას მოგცემთ დაამატოთ შავი და ნათელი თემების გადამრთველი.
- **State Management კალათისთვის**: დაამატეთ `Zustand` ან `Redux Toolkit`, რათა კალათაში (Cart) და ფავორიტებში პროდუქტის დამატება რეალურ დროში აისახოს მრიცხველებზე.
